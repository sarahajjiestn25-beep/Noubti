## Overview

**QueueFlow** is a production-ready web application that enables establishments (medical clinics, government agencies, service centers, training centers, salons, etc.) to digitalize their client reception process.

Clients scan a **QR Code** displayed at the service desk, fill in a short registration form, and instantly receive a **numbered ticket**. Staff can then manage the live queue from a dedicated admin dashboard — calling the next client, updating statuses, and monitoring activity in real time.

The application is fully **multi-service**, **multi-role**, and **white-label configurable** via a SuperAdmin panel (custom logo, colors, application name, contact details).

## Features

### MVP (v1.0)

| Feature | Description |
|---|---|
| Authentication & RBAC | Secure login with 4 distinct roles (SuperAdmin, Admin, Service Manager, Client) |
| Multi-Service Management | Create, activate, or deactivate any number of services |
| QR Code Generation | Each service has a unique, dynamically generated QR Code |
| Client Self-Registration | Client fills a form after scanning; no account required |
| Automatic Ticket Attribution | Sequential numbering per service, per day |
| Queue Status Management | Status lifecycle: `En attente` → `En cours` → `Terminé` / `Annulé` |
| Estimated Wait Time | Dynamic countdown visible to the client on their ticket screen |
| Admin Dashboard | Overview of daily reservations, queue depth, and service activity |
| Global App Configuration | SuperAdmin can customize name, logo, colors, and contact info |

### Future Roadmap (v2.0+)

- Email & SMS notifications on status change
- Real-time display screen (TV board) via Vue.js 3 websocket
- Advanced statistics & reporting (daily/monthly/per-service)
- Mobile application (React Native or Flutter)
- Multi-language support (FR / AR / EN)

## Prerequisites

Before you begin, ensure the following are installed on your machine:

- PHP >= 8.1 with extensions: `mbstring`, `openssl`, `pdo_mysql`, `xml`, `gd`
- Composer >= 2.x
- Node.js >= 18.x & NPM
- MySQL >= 8.x
- Git

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/your-username/queueflow.git
cd queueflow
```

### 2. Install PHP dependencies

```bash
composer install
```

### 3. Install JS dependencies & compile assets

```bash
npm install
npm run dev
```

### 4. Copy and configure the environment file

```bash
cp .env.example .env
php artisan key:generate
```

---

## Environment Configuration

Open `.env` and update the following values:

```env
APP_NAME="QueueFlow"
APP_ENV=local
APP_KEY=         # Auto-generated by php artisan key:generate
APP_DEBUG=true
APP_URL=http://localhost:8000

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=queueflow_db
DB_USERNAME=root
DB_PASSWORD=your_password
```

## Database Setup

```bash
# Create the database first in MySQL
mysql -u root -p -e "CREATE DATABASE queueflow_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# Run migrations
php artisan migrate

# Seed default roles, super admin, and configuration
php artisan db:seed
```

## User Roles

## API Architecture

The application follows an **MVC + REST API** pattern:

- Web routes (`routes/web.php`) serve server-rendered Blade views for the admin panel
- API routes (`routes/api.php`) expose JSON endpoints for real-time ticket status polling (jQuery AJAX) and the Vue.js queue display component
- Middleware enforces role-based access on every protected route
